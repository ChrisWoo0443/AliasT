    **1. Add `_alias_resolve_style` function** -- Insert it in section 4 (Ghost text rendering), before `_alias_show_ghost`. This function resolves style presets to concrete highlight specs (per D-07).

    ```zsh
    # Resolve style preset to a highlight spec.
    # Priority: ALIAS_SUGGESTION_HIGHLIGHT (custom) > ALIAS_SUGGESTION_STYLE (preset) > default (dark)
    _alias_resolve_style() {
      # Custom override always wins
      if [[ -n "$ALIAS_SUGGESTION_HIGHLIGHT" ]]; then
        echo "$ALIAS_SUGGESTION_HIGHLIGHT"
        return
      fi

      # Named presets using hex colors (terminal-palette-independent)
      case "${ALIAS_SUGGESTION_STYLE:-dark}" in
        dark)       echo "fg=#666666" ;;
        light)      echo "fg=#999999" ;;
        solarized)  echo "fg=#586e75" ;;
        *)          echo "fg=#666666" ;;
      esac
    }
    ```

    Key design decisions:
    - Default is "dark" preset (most common terminal theme), NOT "fg=8" which is invisible on Solarized
    - Presets use hex colors (`fg=#RRGGBB`) to be palette-independent (per research pitfall 1)
    - Custom `ALIAS_SUGGESTION_HIGHLIGHT` always overrides presets (per D-07: "Custom ALIAS_SUGGESTION_HIGHLIGHT still works and overrides presets")
    - Unknown preset names fall through to dark preset colors (safe default)
    - `dark=#666666` = medium gray visible on dark backgrounds
    - `light=#999999` = darker gray visible on light backgrounds
    - `solarized=#586e75` = Solarized base01 (visible on both solarized dark and light)
