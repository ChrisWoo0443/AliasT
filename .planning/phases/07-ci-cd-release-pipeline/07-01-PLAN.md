Create `.github/workflows/` directory and write `release.yml` with this exact content:

```yaml
name: Release

on:
  push:
    tags: ["v*.*.*"]

permissions:
  contents: write

jobs:
  build:
    strategy:
      matrix:
        include:
          - target: aarch64-apple-darwin
            runner: macos-15
          - target: x86_64-apple-darwin
            runner: macos-15-intel
    runs-on: ${{ matrix.runner }}
    steps:
      - uses: actions/checkout@v4

      - uses: dtolnay/rust-toolchain@stable

      - uses: Swatinem/rust-cache@v2
        with:
          key: ${{ matrix.target }}

      - name: Build
        run: cargo build --release -p aliast-daemon

      - name: Verify binary architecture
        run: file target/release/aliast-daemon

      - name: Upload binary
        uses: actions/upload-artifact@v4
        with:
          name: aliast-daemon-${{ matrix.target }}
          path: target/release/aliast-daemon

  release:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/download-artifact@v4
        with:
          path: artifacts

      - name: Package release assets
        run: |
          mkdir -p release

          for arch in aarch64-apple-darwin x86_64-apple-darwin; do
            tar -czf "release/aliast-daemon-${arch}.tar.gz" \
              -C "artifacts/aliast-daemon-${arch}" aliast-daemon
          done

          cp plugin/aliast.plugin.zsh release/

          cd release
          sha256sum * > SHA256SUMS

      - name: Create GitHub Release
        uses: softprops/action-gh-release@v2
        with:
          files: release/*
```

Key design decisions and why:
- Per D-02: Hand-written workflow, not cargo-dist. Uses macos-15 (ARM64) and macos-15-intel (x86_64) native runners -- NOT the deprecated macos-13.
- Per D-03: Produces `aliast-daemon-aarch64-apple-darwin.tar.gz`, `aliast-daemon-x86_64-apple-darwin.tar.gz`, `aliast.plugin.zsh`, and `SHA256SUMS`.
- Per D-04: No changelog body -- softprops/action-gh-release creates a minimal release with just the tag name.
- Per D-01: Workflow is repo-agnostic (uses GITHUB_TOKEN from runner context) -- works on ChrisWoo0443/AliasT.
- No `--target` flag on `cargo build` because each runner natively produces the correct architecture. Binary lands in `target/release/` (not `target/<triple>/release/`).
- `sha256sum` runs on ubuntu-latest where it is the standard tool.
- `permissions: contents: write` explicitly set to avoid 403 errors on release creation.
- Unique artifact names via `${{ matrix.target }}` suffix to satisfy upload-artifact v4 requirements.
- Architecture verification step (`file` command) for debugging if wrong-arch binaries are ever produced.
- Flat tarball structure -- `tar -xzf` extracts `aliast-daemon` directly into cwd, which is what the Homebrew formula in Phase 8 expects.
