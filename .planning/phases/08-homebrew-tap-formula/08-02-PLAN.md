**Step 1 — Download SHA256SUMS from the release:**

Determine the version tag from the latest release:
```bash
VERSION=$(gh release view --repo ChrisWoo0443/AliasT --json tagName -q '.tagName' | sed 's/^v//')
echo "Release version: $VERSION"
```

Download SHA256SUMS:
```bash
curl -sL "https://github.com/ChrisWoo0443/AliasT/releases/download/v${VERSION}/SHA256SUMS" -o /tmp/aliast-sha256sums.txt
cat /tmp/aliast-sha256sums.txt
```

**Step 2 — Extract individual SHA256 values:**
```bash
SHA_ARM=$(grep 'aliast-daemon-aarch64-apple-darwin.tar.gz' /tmp/aliast-sha256sums.txt | awk '{print $1}')
SHA_X86=$(grep 'aliast-daemon-x86_64-apple-darwin.tar.gz' /tmp/aliast-sha256sums.txt | awk '{print $1}')
SHA_PLUGIN=$(grep 'aliast.plugin.zsh' /tmp/aliast-sha256sums.txt | awk '{print $1}')

echo "ARM64:  $SHA_ARM"
echo "x86_64: $SHA_X86"
echo "Plugin: $SHA_PLUGIN"
```

Verify all three are 64-character hex strings. If any is empty, the SHA256SUMS file format may differ from expectations — inspect the file and extract manually.

**Step 3 — Update Formula/aliast.rb:**
```bash
cd /tmp/homebrew-aliast
git pull origin main
```

Replace the three placeholder values AND update the version:
- Replace `PLACEHOLDER_SHA256_ARM64` with the actual `$SHA_ARM` value
- Replace `PLACEHOLDER_SHA256_X86_64` with the actual `$SHA_X86` value
- Replace `PLACEHOLDER_SHA256_PLUGIN` with the actual `$SHA_PLUGIN` value
- Update `version "0.1.0"` to `version "$VERSION"` if the release tag differs from 0.1.0

Use `sd` (preferred) or `sed` for the replacements:
```bash
sd 'PLACEHOLDER_SHA256_ARM64' "$SHA_ARM" Formula/aliast.rb
sd 'PLACEHOLDER_SHA256_X86_64' "$SHA_X86" Formula/aliast.rb
sd 'PLACEHOLDER_SHA256_PLUGIN' "$SHA_PLUGIN" Formula/aliast.rb
```

**Step 4 — Verify no placeholders remain:**
```bash
grep -c 'PLACEHOLDER' Formula/aliast.rb
# Must return 0
```

**Step 5 — Verify SHA256 values are 64-char hex strings:**
```bash
grep -oP '[a-f0-9]{64}' Formula/aliast.rb | wc -l
# Must return 3 (one per sha256 field)
```

**Step 6 — Commit and push:**
```bash
cd /tmp/homebrew-aliast
git add Formula/aliast.rb
git commit -m "feat: populate SHA256 values from v${VERSION} release"
git push origin main
```

**Step 7 — Local brew install test (before the human-verify checkpoint):**

Clear any stale local state:
```bash
brew untap ChrisWoo0443/aliast 2>/dev/null || true
brew uninstall aliast 2>/dev/null || true
brew cleanup -s aliast 2>/dev/null || true
```

Install from the tap:
```bash
brew tap ChrisWoo0443/aliast
brew install aliast
```

Verify installation:
```bash
which aliast-daemon
aliast-daemon --version
ls -la "$(brew --prefix)/share/aliast/aliast.plugin.zsh"
brew info aliast
```

If `brew install` fails with SHA256 mismatch:
1. Check if `SHA256SUMS` was read correctly: `cat /tmp/aliast-sha256sums.txt`
2. Clear Homebrew cache: `rm -rf ~/Library/Caches/Homebrew/downloads/*aliast*`
3. Re-download and verify: `curl -sL <url> | shasum -a 256`
4. Update the formula with the correct SHA and retry

**Anti-pattern guards:**
- NEVER compute SHA256 from local build artifacts — always from the published release
- NEVER leave PLACEHOLDER values in the committed formula
- ALWAYS `brew uninstall` between iterations to avoid stale Cellar state (Pitfall 7)
