Wire HistoryStore into the daemon: initialize SQLite on startup, auto-import zsh history when DB is empty, pass shared store to connection handlers, dispatch Record requests, and update Complete to use history-based suggest().

Purpose: The data layer (Plan 01) exists but is not connected to the running daemon. This plan makes the daemon actually serve real suggestions from history and accept new command recordings.

Output: Daemon that opens SQLite on startup, imports history once, serves prefix-matched suggestions, and records new commands.

  <behavior>
    - Daemon creates ~/.local/share/alias/history.db on startup (per D-10)
    - If DB is empty (count() == 0), daemon reads ~/.zsh_history, parses it, and imports entries (per D-01)
    - If ~/.zsh_history does not exist, daemon starts with empty DB (no error)
    - HistoryStore is wrapped in Arc<std::sync::Mutex<HistoryStore>> and passed to run_server
    - run_server passes the shared store to each handle_connection call
    - handle_connection accepts the shared store and uses it in dispatch_request
    - Complete request: locks store, calls alias_core::suggest(&store, &buf), returns Suggestion
    - Record request: locks store, calls store.record_command(cmd, timestamp, cwd), returns Ack
    - Existing Ping/Pong behavior is unchanged
  </behavior>
  <action>
    **Step 1: Update Cargo.toml.**
    No new dependencies needed -- alias-core already has rusqlite. The daemon depends on alias-core which re-exports HistoryStore.

    **Step 2: Update main.rs.**
    In the `Commands::Start` handler, after resolving socket_path and before spawning the server:
    1. Resolve DB path: `~/.local/share/alias/history.db` using the same `directories::BaseDirs` pattern already used for log_dir (per D-10). Create the directory if it does not exist.
    2. Open HistoryStore: `let store = alias_core::history::HistoryStore::open(&db_path)?;`
    3. Auto-import check (per D-01): If `store.count()? == 0`, read `~/.zsh_history` (expand `$HOME/.zsh_history`), parse with `alias_core::history::parse_history_file(&content)`, assign monotonically increasing synthetic timestamps (line index + 1) to entries with None timestamps (per Research open question 3), then call `store.import_entries(&entries)`. Log the import count with tracing::info!. If ~/.zsh_history does not exist, log a debug message and continue.
    4. Wrap store: `let shared_store = Arc::new(std::sync::Mutex::new(store));`
    5. Pass `shared_store.clone()` to `run_server`.

    **Step 3: Update server.rs.**
    Change `run_server` signature to accept `store: Arc<Mutex<HistoryStore>>`:
    ```rust
    pub async fn run_server(
        socket_path: &Path,
        cancel_token: CancellationToken,
        store: Arc<Mutex<alias_core::history::HistoryStore>>,
    ) -> Result<()>
    ```
    In the accept loop, clone `store` for each connection and pass to `handle_connection`:
    ```rust
    let conn_store = store.clone();
    tokio::spawn(async move {
        if let Err(err) = handle_connection(stream, child_token, conn_store).await {
            tracing::error!("Connection handler error: {err}");
        }
    });
    ```

    **Step 4: Update connection.rs.**
    Change `handle_connection` signature to accept `store: Arc<Mutex<alias_core::history::HistoryStore>>`.
    Change `dispatch_request` to accept `store: &Arc<Mutex<alias_core::history::HistoryStore>>`.

    In dispatch_request:
    - `Request::Complete { id, buf, cur: _ }`: Lock the mutex, call `alias_core::suggest(&store_guard, &buf)`, unlock. Return `Response::Suggestion { id, text }`.
    - `Request::Record { id, cmd, cwd }`: Lock the mutex, call `store_guard.record_command(&cmd, current_unix_timestamp, &cwd)`. Use `std::time::SystemTime::now().duration_since(UNIX_EPOCH).unwrap().as_secs() as i64` for the timestamp. Return `Response::Ack { id }`.
    - `Request::Ping { id }`: Unchanged.

    Use `std::sync::Mutex` (not tokio::sync::Mutex) since the lock is held only during synchronous SQLite operations that complete in microseconds (per Research Pattern 2). This avoids async overhead.

    **Step 5: Verify compilation.**
    `cargo build --workspace` must succeed.
